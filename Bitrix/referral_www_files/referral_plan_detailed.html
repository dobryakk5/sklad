CRM — HelperRest.php
Приём данных от 1С, создание/обновление реферала
▶
Что добавить
Метод addReferralFrom1C($params) — создаёт новую запись реферала в CRM. По аналогии с addInvoiceFrom1C(): ищет существующего пользователя, создаёт CRM-сущность (смарт-процесс или доп. поле контакта), пишет диагностический лог.

Поиск дублей по CONTACT_ID + UF_REF_CODE перед созданием — чтобы не множить записи при повторных вызовах.

Метод Exchange::AddReferral()
Ищет пользователя сайта по USER_ID или EMAIL.
Проверяет существование записи в HL-блоке ReferralProfile по USER_ID.
Создаёт или обновляет запись: UF_REF_CODE, UF_STATUS, UF_BONUS_AMOUNT, UF_REFERRAL_URL, UF_DATE_CREATE.
Возвращает JSON-ответ с результатом — CRM логирует успех/ошибку.

Фаза 4
HL-блок ReferralProfile
Создание новой сущности хранения на сайте
▶
Создать HL-блок через админку или миграцию
Название таблицы: b_hl_referral_profile. Название объекта: ReferralProfile.
Поля HL-блока
ПолеТипНазначение
UF_USER_IDintegerID пользователя сайта (уникальный)
UF_REF_CODEstring(32)Реферальный код — ALFA-XXXXX
UF_STATUSstring(16)ACTIVE / INACTIVE / PENDING
UF_BONUS_AMOUNTdecimalСумма бонуса к начислению (из 1С)
UF_REFERRAL_URLstring(255)Полная ссылка с utm/ref-параметром
UF_DATE_CREATEdatetimeДата генерации кода в 1С
UF_REFERRAL_COUNTintegerКол-во привлечённых клиентов (из 1С)
UF_DATE_UPDATEdatetimeПоследнее обновление из CRM
UF_USER_ID — уникальный индекс. Один пользователь = одна запись. Все обновления идут через Update, не через дублирование.

Фаза 6
Отслеживание перехода по реф-ссылке
Запись REF_CODE при регистрации нового клиента
▶
Сценарий
Новый клиент переходит по ссылке alfasklad.ru/?ref=ALFA-XXXXX. Сайт сохраняет GET-параметр ref в сессию / cookie.
При регистрации нового пользователя — обработчик OnAfterUserRegister считывает REF_CODE из сессии и записывает в доп. поле пользователя UF_REFERRED_BY.
Далее 1С при следующей синхронизации получает UF_REFERRED_BY через существующий REST-метод enum.addDealFromSite или отдельный вызов, и начисляет бонус рефереру.
Логика в init.php
// Сохраняем ref в сессию при заходе на сайт AddEventHandler('main', 'OnPageStart', function() { if (!empty($_GET['ref'])) { $_SESSION['REF_CODE'] = htmlspecialchars($_GET['ref']); } }); // Записываем при регистрации AddEventHandler('main', 'OnAfterUserRegister', function(&$arFields) { if (!empty($_SESSION['REF_CODE'])) { CUser::SetUserField($arFields['USER_ID'], 'UF_REFERRED_BY', $_SESSION['REF_CODE']); unset($_SESSION['REF_CODE']); } });
Доп. поле пользователя
Создать пользовательское поле UF_REFERRED_BY типа string в сущности «Пользователи» — хранит REF_CODE того, кто привёл клиента.
